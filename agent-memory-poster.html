THE LAYERS OF AGENT MEMORY

How agents remember — working, episodic, semantic, procedural, meta
CTF Farooq Ali
· AI SDLC course · explainer series
v2026.06
01
Why memory at all — a bare LLM is stateless

Every turn starts from zero. The model only sees what's inside its context window right now — close the session and it's all gone.

No persistence

Yesterday's decisions, your name, what worked — forgotten by default.

No learning

It can't get better at your task over time. Same mistakes, every time.

With memory

The agent can persist, recall, and compound — and not repeat itself.

Memory is the layer that turns a stateless model into an agent that accumulates.

02
The runtime — one turn — long-term layers are READ before, WRITTEN after

Working memory holds the live turn; the long-term layers are retrieved into the prompt at planning time and updated once the turn is done.

03
L1 · Working memory — now

The current turn's context — the prompt, the recent steps, the scratch state the agent is reasoning over right this moment.

bounded by the context window resets at session end

Without it: the agent can't hold a coherent train of thought across a single task.

04
L2 · Episodic memory — what happened

Cross-session traces of past interactions — what was asked, what was done, how it went. Append-only; retrieved at planning.

append-only log retrieved by relevance

Without it: every session feels like the first — no continuity.

05
L3 · Semantic memory — what's true

An entity store of typed facts — people, places, projects, preferences (who / what), queried by ID or by similarity.

typed facts lookup by ID or similarity

Without it: the agent re-asks for things it was already told.

06
L4 · Procedural memory — how to

A skill registry of learned procedures and how-tos — reusable playbooks the agent has acquired, indexed by task type.

skills & playbooks indexed by task type

Without it: the agent re-derives the same method from scratch each time.

07
L5 · Meta-memory — the control plane

The layer that manages the other layers: archive, dedupe, deprecate, decay — deciding what to keep, merge, age out, or forget.

dedupe & resolve decay & deprecate

Without it: the stack rots — contradictions pile up and stale facts mislead.

08
The stack — five layers, each one feeds the next

Read top-down it's a maturity stack: working remembers now; episodic, semantic and procedural let the agent compound; meta keeps it all clean.

09
Read vs write

READ — at planning (memory feeds the prompt):

recall traces→ fetch entities→ load skills

WRITE — after the turn (memory is updated):

append trace→ extract entities→ reflect on procedures

Working memory is live the whole turn; the long-term layers bracket it — read before, write after.

10

Five memory systems in cognitive science map onto five layers in the architecture.

11
What nobody tells you
⚖️
Contradictions — without dedup + entity resolution, retrieval serves two conflicting "facts" at once.
🪙
Context overload — unbounded memory floods the window — more tokens, more cost, worse focus.
⌛
Stale memories — yesterday's truth is today's bug. Old facts need decay, not permanence.
🎯
Retrieval gates everything — if the wrong slice is pulled in, every later layer reasons on the wrong basis.
🔒
Privacy / PII — long-term stores quietly accumulate personal data — governance and deletion matter.
🧭
It's a pattern, not a product — "5 layers" is a useful model; real systems split or merge these differently.
12
The takeaway
Five memory systems in cognitive science ≈ five architectural layers in production AI. Working remembers now — the rest let the agent compound over time.
Working→ Episodic→ Semantic→ Procedural→ Meta
