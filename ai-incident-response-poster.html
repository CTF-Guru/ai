01
What it is — a plan, not a panic

AI incident response is the process for detecting, containing, investigating, fixing, and preventing harmful AI system behavior.

Same muscle as classic SecOps incident response — but the failure modes are AI-shaped: bad outputs, leaked data, misused tools, injected instructions, degraded retrieval, runaway cost. You plan the response before the model ships, not during the outage.

02
The response loop — an alert routes through containment to recovery, then prevention
03
Track the status — one incident, four states it moves through
⬤ DETECTED→ ⬤ CONTAINED→ ⬤ FIXED→ ⬤ VERIFIED

Don't close on fixed. Verified means an eval reproduces the old failure and now passes — proof, not a vibe.

04
What an AI incident looks like — six you should expect

🧠 Hallucinated answer

Model states a confident, wrong answer on a critical path — medical, legal, financial.

💧 Data leakage

Secrets, PII, or another tenant's data surface in an output or a log.

💉 Prompt injection

A retrieved doc or tool result hijacks the agent's instructions.

🛠️ Unauthorized tool action

Agent calls a tool it shouldn't — sends, deletes, pays, deploys.

📚 Bad retrieval / citation

Wrong or stale source retrieved; the citation doesn't support the claim.

💸 Cost spike or loop

A runaway loop or fan-out burns tokens and budget with no progress.

05
Containment actions
🔌
Disable the tool — kill the dangerous capability, not the whole app.
↩️
Roll back — revert the prompt, model version, or config to last-good.
🚫
Block the source — quarantine the poisoned doc or feed.
✋
Require approval — force a human gate on risky actions.
🐢
Rate-limit / pause — throttle or freeze the workflow.
📣
Notify owners — page the people who can decide.
06
The common failure
🕳️
No traces, no evals — without them, teams can't tell what failed and can't prove it's fixed. The incident just goes quiet.
📜
No runbook — "who can disable this tool at 2am?" gets answered live, badly.
🔁
No verify step — closed as "fixed," reopens next week. Same bug, no test.

Traces + evals are the difference between a fix and a guess.

08
The takeaway
Plan AI rollback, tool shutdown, and source quarantine before production launch — so the response is a runbook, not an improvisation.
Write the runbook→ Wire kill-switches→ Keep traces + evals→ Drill it
09
Where it sits in the AI SDLC — the loop after launch
Monitor→ Incident Response→ Improve→ Governance

Monitoring catches it · response contains and fixes it · improvement hardens the system · governance turns each incident into a policy so it can't recur unnoticed.
