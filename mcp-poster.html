03
What MCP actually is

Three parts. Built on JSON-RPC 2.0.

🖥️ Host

The AI app. Claude Code, Cursor, a desktop client.

🔌 Client

Lives inside the host. One per server, 1:1.

🗄️ Server

Exposes capabilities. Local process or remote.

Host contains Clients · each Client talks to one Server.

04
The three primitives

Tools

Model-controlled — functions it can call.search_db, send_email

Resources

App-controlled — data it can read.files, schemas

Prompts

User-controlled — reusable templates./summarize

Most cheat-sheets skip this. It's the whole mental model.

05
How it travels
⚡
stdio — local subprocess. Fastest, no network. Filesystem, git.
🌐
Streamable HTTP — remote servers. Replaced HTTP+SSE in 2025. Cloud services.
06
Lifecycle
initialize→ capabilities→ initialized
tools/list→ tools/call→ shutdown

Handshake first, then the operation phase repeats.

07
On the wire — every call is one JSON-RPC message
// the model asks the server to run a tool
{ "jsonrpc": "2.0", "id": 42, "method": "tools/call",
  "params": { "name": "query_database",
    "arguments": { "sql": "SELECT * FROM users LIMIT 10" } } }

Request → response, every time. That's the entire protocol surface.

What nobody tells you
🪙
Token tax — every tool schema rides in the context. 50 tools ≈ 50 schemas, every turn.
🔑
OAuth sprawl — secret rotation across N remote servers gets real.
🧰
Tool sprawl — more tools = more surface to mis-pick. Curate.
📐
Schema drift — server v2 changes a field; agents break silently.
🔭
Observability gap — no standard tracing yet. Log across calls.
