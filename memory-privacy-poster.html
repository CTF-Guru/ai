MEMORY PRIVACY

Remember only what you should — consent, transparency, control & retention for agent memory

01
What memory privacy is — governance, not a feature toggle

Memory privacy controls what an agent may remember, why it remembers it, how long it keeps it, and how users can control it.

Persistent memory makes agents feel personal — but every fact it keeps is data you now hold, govern, and can leak. Personalization is a privilege the user grants, not a default the agent assumes.

02
The memory vault — nothing is stored until it clears four gates

Pass all four → store. Fail any one → ask, scope, redact, or drop.

03
Classify every candidate memory — five outcomes, not just yes/no
✓ Store· ? Ask first· ⏱ Store temporarily· ✕ Do not store· ▦ Redact / protect

✓ Store · ? Ask first

Store a stated preference the user opted into. Ask first when it's useful but consent is unclear.

⏱ Store temporarily

Keep only for the task or session, then auto-expire. The default for transient context.

✕ Do not store · ▦ Redact

Drop what should never persist. Redact when part is needed but identifiers are not.

04
Never casually remember
🔑
Secrets & credentials — passwords, API keys, tokens. Never persist.
🪪
Regulated personal data — PII under GDPR / HIPAA / CCPA.
🏥
Sensitive health / financial / legal — protect or avoid entirely.
🏢
Confidential business data — contracts, internal plans, customer lists.
❓
Unverified claims about people — storing a guess turns it into a "fact".
05
The common failure
🕳️
Silent memory collection — the agent quietly remembers things the user never agreed to and cannot see.
⚖️
Trust & compliance fallout — invisible retention = surprise, broken trust, and regulatory exposure.
🔭
No audit trail — if you can't show what was stored and why, you can't defend it.

If memory is invisible, it is a liability — not a feature.

06
Where it lives in the AI SDLC — privacy is designed in, then watched
Product Design→ Privacy Review→ Build→ Monitor

Decide what may be remembered at design time · gate it in a privacy/governance review · enforce consent & expiry in the build · audit live memory in monitoring.

07
The takeaway
Good memory isn't "remember everything." It's remember only what you should — and prove you can show, change, scope, and forget it.
Visible→ Editable→ Scoped→ Expirable
