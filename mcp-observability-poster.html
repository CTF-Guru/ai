01
What MCP observability is — the flight recorder for tool-using AI

Observability records what the AI requested, which tool was called, what happened, and whether the action was safe. In production you can't just see the final answer — you need the trail behind every tool call an agent makes.

🔍 See the ask

The model's plan and the tool it chose — before anything runs.

🛡️ See the guard

Validation + approval checks — what passed, what was blocked.

📊 See the result

Output summary, latency, cost, errors — evidence after the fact.

02
The request trace — one tool call, traced end to end

The trace is the product. Plan → select → validate → approve → run → result — captured as it happens, replayable after.

03
What to capture
🆔
User / session ID — who, in which session, on whose behalf.
🔧
Tool name + version — which capability ran, which schema version.
↔️
Input / output summaries — arguments in, result out — summarised, not raw.
✅
Approval state — auto-allowed, human-approved, or blocked.
⏱️
Latency & errors — how long, what failed, was it retried.
🪙
Cost / token usage — tokens and spend, where available.
🚩
Security flags — anything the policy layer wanted to surface.
04
Privacy warning — logs become a liability

Capturing everything is its own risk. Do not log raw:

🔑
Secrets & credentials — API keys, tokens, passwords. Never in plaintext logs.
🏛️
Regulated data — PII, health, financial, legal — policy decides retention.
📦
Full sensitive payloads — summarise & redact; don't dump the whole body.

Trace the shape, not the secret. Redaction is part of the logger, not an afterthought.

05
Why it matters

🐞 Debug

Reconstruct exactly which call went wrong, and why.

⚖️ Comply

Show auditors who did what, when, and under which approval.

🚨 Respond

During an incident, the trace is your timeline of record.

And a fourth: improve — traces are the data you tune quality and tool-choice against.

06
The common failure
🕳️
Log nothing — the agent acts, something breaks, and there's no trail to follow. A blind spot you only notice in the incident.
🔓
Log everything raw — full payloads and secrets land in plaintext — now the log itself is the breach.

The job is the middle path: capture enough to govern, redact enough to be safe.

07
Reading the status chips — five outcomes every step can report

allowedValidation passed; safe to proceedinput validationProceed; keep the record
blockedPolicy denied the actionapproval / permissionAlert; surface to a human
approvedA human (or rule) authorised itapproval checkAttribute the approver

retriedRe-attempted after a failureexecution / resultWatch for retry storms

Status is the spine of the trace — it's how a 10,000-row log becomes a filter you can actually search.

08
The takeaway
You can't govern, debug, or improve tool-using AI you can't trace. Observability isn't an add-on — it's how an agent earns the right to act.
Trace every call→ Record the status→ Redact the sensitive→ Govern & improve
09
Where it lives in the AI SDLC — Deploy → Monitor → Audit → Improve
Deploy→ Monitor→ Audit→ Improve

Ship the tool · watch every call · keep the evidence trail · feed traces back into quality. The loop closes here.
